# jaeger-mcp-server (Rust)

An [MCP](https://modelcontextprotocol.io) server for [Jaeger](https://www.jaegertracing.io/) written in Rust, using the [`rmcp`](https://crates.io/crates/rmcp) SDK and the Jaeger v3 HTTP API.

Feature-parity port of the Node.js `jaeger-mcp-server`, with the following fixes baked in:

- `find-traces` now sends `start_time_min` / `start_time_max` as RFC 3339 strings (was raw millis ‚Üí HTTP 400).
- `duration_min` / `duration_max` are serialized as `google.protobuf.Duration` (`1.500s`), not `100ms`.
- `attributes` are actually forwarded to the API (`query.attributes[key]=value` for each entry).
- `find-traces` defaults to `search_depth = 20` when omitted, so requests don't page the whole store.
- Auth is set once on the HTTP client rather than per request.

Only the **HTTP** protocol is implemented in this initial version. gRPC (`tonic` + `prost`) is planned as a follow-up.

## Build

```powershell
cargo build --release
```

Output: `target/release/jaeger-mcp-server` (`.exe` on Windows) ‚Äî a single static binary.

## Run locally

```powershell
$env:JAEGER_URL = "http://localhost"
$env:JAEGER_PORT = "16686"
.\target\release\jaeger-mcp-server.exe
```

The process reads MCP JSON-RPC on stdin and writes it to stdout. All logs go to stderr.

## Environment variables

| Variable | Required | Default | Description |
| --- | --- | --- | --- |
| `JAEGER_URL` | yes | ‚Äî | Base URL of Jaeger (`http://host` or `https://host`). Scheme optional. |
| `JAEGER_PORT` | no | `16686` (or `443` for `https`) | Jaeger v3 HTTP API port. |
| `JAEGER_AUTHORIZATION_HEADER` | no | ‚Äî | Value for the `Authorization` header (e.g. `Basic <base64>` or `Bearer <token>`). |
| `RUST_LOG` | no | `info` | Log level (`error`, `warn`, `info`, `debug`, `trace`). |

## VS Code / Copilot MCP config

```jsonc
{
  "servers": {
    "jaeger-mcp-server": {
      "type": "stdio",
      "command": "c:\\path\\to\\jaeger-mcp-server-rs\\target\\release\\jaeger-mcp-server.exe",
      "env": {
        "JAEGER_URL": "http://localhost",
        "JAEGER_PORT": "16686"
      }
    }
  }
}
```

Or, after `cargo install --path .`, drop the absolute path:

```jsonc
"command": "jaeger-mcp-server"
```

## Tools exposed

- `get_services` ‚Äî list all known service names.
- `get_operations` ‚Äî list operations for a service, optionally filtered by span kind.
- `get_trace` ‚Äî fetch a trace by 32-char hex id, with optional start/end time bounds.
- `find_traces` ‚Äî search traces by service, operation, attributes, time window, and duration.

## Distribution

Recommended setup once you push this to GitHub:

1. **`cargo-dist`** ‚Äî generates a GitHub Actions release workflow that publishes prebuilt binaries for Windows, macOS (Intel + Apple Silicon), and Linux (x64 + arm64) on every `git tag v*`.
2. **`cargo binstall`** ‚Äî free once step 1 is in place.
3. **npm shim** ‚Äî so `npx -y jaeger-mcp-server` still works for MCP clients that document npx-based installs. `cargo dist init --installer=npm` scaffolds it.
4. **Homebrew tap / Scoop bucket** ‚Äî also generated by `cargo-dist`.

See the top-level project notes for details.
